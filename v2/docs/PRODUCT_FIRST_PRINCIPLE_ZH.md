# 产品第一原则（必须遵守）

更新时间：2026-02-23

## 原则 1：非开发用户优先

不会开发的人，也能更方便地自动化生成脚本。

## 原则 2：开源优先复用

在实现复杂能力前，先调研并复用成熟开源方案；禁止闭门重复造轮子。

## 强制验收标准

1. 不要求用户手写代码。
2. 不要求用户手动找 `xpath/css/framePath`。
3. 主流程可通过可视化完成，并支持一键生成脚本。
4. 所有核心按钮、提示、报错信息使用中文，且可直接指导下一步操作。
5. 复杂页面可用：至少支持嵌套 iframe、弹窗、新标签页场景。
6. 每个复杂能力在开发前必须有“开源对标清单”（来源、可复用点、取舍理由）。

## 实施优先级（按顺序）

1. 先保证元素拾取稳定（含嵌套 iframe 与 fallback）。
2. 再做脚本生成稳定（自动携带 framePath/frameLocator）。
3. 最后优化交互体验（操作条、提示气泡、快捷键、错误恢复）。

## 开源调研门禁

1. 至少调研 3 个可用开源项目或官方实现。
2. 明确记录可直接复用的模块，不只记录产品名。
3. 对每个候选写明许可证与接入风险。
4. 未完成调研记录，不进入编码阶段。

## 禁止偏离

如果某项实现不能让非开发用户更省事，即使技术上可行，也不作为完成标准。
