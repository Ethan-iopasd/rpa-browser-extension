# UI 重构计划（Dify 风格，多页面拆分）

最后更新：2026-02-21

## 1. 背景与目标

当前单页面承载了编排、运行、调度、版本、录制导入等多块内容，信息密度过高，导致：

1. 新用户学习成本高。
2. 关键任务路径不清晰。
3. 运行排障与流程编辑相互干扰。

本计划目标：

1. 按任务流拆分页面，降低每页复杂度。
2. 保留 Dify 风格画布体验。
3. 引入 RPA 专用“步骤视图”，兼顾新手与高级用户。

## 2. 信息架构（目标页）

1. `/dashboard`：总览（成功率、失败告警、待执行任务）。
2. `/flows`：流程列表（检索、标签、状态）。
3. `/flows/:flowId/editor`：流程编辑（画布 + 属性面板）。
4. `/runs`：运行中心列表（筛选、导出、失败聚类）。
5. `/runs/:runId`：运行详情（控制台、事件、错误定位）。
6. `/tasks`：任务列表（手动/定时/批量）。
7. `/tasks/:taskId`：任务详情（调度配置、历史、重跑）。
8. `/security/credentials`：凭据管理（加密与最小展示）。
9. `/settings`：系统配置（并发、阈值、调度开关）。

## 3. 设计原则

1. 编辑页只负责“编排”，不塞运行日志与任务大表。
2. 运行控制台独立详情页，支持长日志阅读与定位。
3. 定时任务独立页，避免与画布配置混杂。
4. 默认简洁模式，高级参数折叠。
5. 全中文文案统一词表（同一概念只用一种说法）。

## 4. 分阶段执行

### 阶段 A：应用骨架与路由（1-2 天）
状态：已完成（2026-02-21）。

1. 引入页面级路由与统一布局（侧边导航 + 顶栏 + 面包屑）。
2. 拆分现有单页状态入口，建立页面容器层。
3. 增加基础页面空态与加载骨架。

交付：

1. 页面可独立访问，导航可达。
2. 不破坏现有 API 调用链。

验收：

1. `flows/runs/tasks/security/settings` 页面路由全部通。

### 阶段 B：流程域拆分（2-3 天）
状态：已完成（2026-02-21）。

1. `flows` 列表页（创建、复制、删除、状态筛选）。
2. `editor` 页仅保留画布、节点属性、变量面板。
3. 专业模式下显示录制导入与版本管理。

交付：

1. 流程创建 -> 编辑 -> 保存 -> 发布闭环。

验收：

1. 编辑页可在不依赖运行页的情况下完成编排与校验。

### 阶段 C：运行中心拆分（2-3 天）
状态：已完成（2026-02-21）。

1. `runs` 列表页：按状态、任务、流程过滤。
2. `run detail`：事件流、错误码、节点定位、日志导出。
3. 支持“从失败运行回跳流程编辑器”。

交付：

1. 运行追踪与排障闭环。

验收：

1. 失败样例可在 3 步内定位到节点。

### 阶段 D：任务中心拆分（2-3 天）
状态：已完成（2026-02-21）。

1. `tasks` 列表页：任务检索、状态切换、批量操作。
2. `task detail`：调度策略、最近运行、失败重跑、告警关联。
3. 任务配置与流程绑定关系可视化。

交付：

1. 任务生命周期管理完整。

验收：

1. 任务创建 -> 启停 -> 重跑 -> 查看历史全流程可用。

### 阶段 E：安全页拆分（1-2 天）
状态：已完成（2026-02-21）。

1. 凭据管理页（新增、查看元数据、审计入口）。
2. 审计日志查询页（按动作/时间过滤）。

交付：

1. 安全能力从系统页中独立。

验收：

1. 凭据与审计查询可独立完成。

### 阶段 F：交互优化（2 天）
状态：已完成（2026-02-21）。

1. 画布连线边上 `+` 快速插入节点。
2. 节点内联关键配置（URL、selector、wait）。
3. 错误就地标记与快捷修复入口。

交付：

1. 常用操作点击次数下降。

验收：

1. 新建一个标准流程（开始->打开->点击->结束）操作步数明显减少。

### 阶段 G：迁移与回归（2 天）
状态：已完成（2026-02-21）。

1. 旧单页入口保留兼容跳转。
2. 前端状态迁移与本地缓存兼容。
3. 回归清单覆盖流程、运行、任务、安全。

交付：

1. 迁移无阻断回归。

验收：

1. 核心功能回归通过，旧入口不 404。

### 阶段 H：可用性打磨（1-2 天）
状态：已完成（2026-02-21）。

1. 全局中文文案统一。
2. 空态/错误态/加载态完善。
3. 新手引导与快捷键说明。

交付：

1. 可对外进行 Beta 体验测试。

验收：

1. 新用户 10 分钟内可完成首次编排与运行。

## 5. 质量门禁

1. `pnpm --filter @rpa/designer typecheck`
2. `pnpm --filter @rpa/designer lint`
3. `pnpm --filter @rpa/designer build`
4. 关键流程录屏回归（创建、运行、调度、排障）

## 6. 首批优先级（建议先做）

1. A + B（路由骨架 + 流程域拆分）
2. C（运行中心拆分）
3. D（任务中心拆分）

说明：先把“编辑/运行/任务”三块分开，体验会立刻改善；E-H 可并行排期。
