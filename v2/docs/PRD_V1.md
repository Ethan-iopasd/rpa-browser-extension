# PRD V1 - RPA Flow

最后更新：2026-02-21

## 1. 产品目标

构建本地优先（local-first）的 RPA 工具，提供可视化拖拽流程设计与稳定的网页自动化执行能力。

## 2. 目标用户

- 自动化重复浏览器流程的个人操作者。
- 需要可复用任务、日志与重试能力的小团队。

## 3. 核心场景

1. 通过节点编排流程并手动触发运行。
2. 录制浏览器交互并转换为节点配置。
3. 配置定时任务并在完成后查看运行日志。

## 4. V1 范围内

- 含节点图与配置面板的流程编辑器。
- 执行前流程校验。
- 面向浏览器任务的本地执行引擎。
- 提供节点级状态与耗时的运行日志。
- 基础定时与重试控制。

## 5. V1 范围外

- 多租户云部署。
- 分布式执行集群。
- 完整桌面应用自动化。

## 6. 功能需求

1. 用户可以创建、编辑、校验并保存流程。
2. 系统支持以下节点类型：
   - Start、End、Navigate、Click、Input、Wait、Extract、If、Loop。
3. 用户可以运行流程并观察实时状态。
4. 系统保存流程版本与运行历史。
5. 失败步骤提供可执行的错误信息。

## 7. 非功能需求

1. 流程运行不得静默失败。
2. 日志必须包含时间戳、节点 ID、节点类型、状态与消息。
3. 数据模型必须版本化（`schemaVersion`）。
4. 核心模块测试覆盖率目标不低于 80%。

## 8. 成功标准

1. 用户在 10 分钟内完成“创建 -> 校验 -> 运行 -> 查看”闭环。
2. 基线冒烟测试中，至少 90% 常见浏览器动作可正常执行。
3. 系统可在 8 小时稳定性测试中保持可重复执行。
