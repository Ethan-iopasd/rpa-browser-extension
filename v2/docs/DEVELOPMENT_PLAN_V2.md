# RPA Flow V2 总开发计划

最后更新：2026-02-21

## 1. 目标

1. 从插件 Demo 重构为可持续迭代的本地优先 RPA 产品。
2. 保留拖拽编排体验，同时提升执行稳定性、可观测性与可维护性。
3. 建立统一 DSL、统一接口契约与统一工程规范。

## 2. 范围（V1）

1. 可视化流程编辑器（节点拖拽、连线、参数配置、流程校验）。
2. 本地执行引擎（浏览器自动化、重试、超时、日志）。
3. 录制扩展（采集操作并导出节点参数）。
4. 任务运行中心（运行历史、步骤日志、失败定位）。

## 3. 分阶段计划

## 阶段 A：需求冻结与技术选型（3-5 天）

1. 产出 PRD V1。
2. 冻结 DSL V1（节点、边、变量、版本号策略）。
3. 确认技术栈与质量门禁。

验收：
1. PRD、架构、里程碑文档评审通过。

状态：已完成。

## 阶段 B：工程骨架搭建（4-6 天）

1. 搭建 V2 Monorepo。
2. 拆分 Designer/API/Agent/Schema 模块。
3. 统一 lint/typecheck/test/build 脚本。
4. 交付本地启动文档与 CI 草案。

验收：
1. 新结构可独立启动。
2. 契约同步脚本可执行。
3. 基线测试可运行。

状态：已完成。

## 阶段 C：DSL 与执行内核（7-10 天）

1. 强化 DSL 校验与版本迁移机制。
2. 实现执行状态机（pending/running/success/failed/canceled）。
3. 完成基础节点执行器（Start/End/Navigate/Click/Input/Wait/Extract/If/Loop）。
4. 补齐失败重试、超时中断、错误码分类。

验收：
1. 标准流程可稳定执行并输出完整结构化日志。

状态：已完成（2026-02-22）。

## 阶段 D：Designer 功能完善（9-12 天）

1. 完成画布、节点面板、变量面板、运行控制台。
2. 流程校验可视化与错误定位。
3. 流程版本管理（草稿/发布/回滚）。

验收：
1. 完成“创建 -> 校验 -> 运行 -> 查看日志 -> 保存版本”闭环。

状态：已完成（2026-02-20）。

## 阶段 E：录制能力与桥接（5-8 天）

1. Recorder 扩展采集 click/input/select 事件，并补充 navigate 事件与上下文。
2. 导出录制数据并可直接回填到 Designer（推送、粘贴、上传）。
3. 处理 iframe 与动态 DOM 的选择器稳定性（主备候选策略）。

验收：
1. 常见网页操作可高成功率生成可执行节点。
2. 形成“录制 -> 导入 -> 校验 -> 运行”闭环。

状态：已完成（2026-02-21）。

## 阶段 F：任务调度与运行中心（6-9 天）

1. 增加任务模型（手动、定时、批量）。
2. 支持并发控制、失败重跑、日志过滤导出。
3. 增加运行统计与告警接口。

验收：
1. 多任务连续运行稳定，结果可追溯。

状态：已完成（2026-02-21）。

## 阶段 G：安全与稳定性加固（5-7 天）

1. 凭据加密与敏感字段脱敏。
2. 长稳测试、异常恢复、性能压测。
3. 发布前高风险场景回归测试。

验收：
1. 关键故障可恢复或可定位，不出现静默失败。

状态：已完成（2026-02-21）。

## 阶段 H：Beta 发布（4-6 天）

1. 安装包与升级策略。
2. 用户操作手册、排障手册、演示模板。
3. Beta 反馈闭环与阻断缺陷修复。

验收：
1. 用户可独立完成安装、编排、执行、排障。

状态：已完成（2026-02-21）。

## 4. 质量门禁（每阶段必须满足）

1. `lint`、`typecheck`、`test`、`build` 通过。
2. 新能力具备最小测试与可观测日志。
3. DSL 变更先改 Schema，再生成类型，再改实现。
4. 文档同步更新。

## 5. 当前里程碑

1. 里程碑 M1（已完成）：阶段 A + 阶段 B。
2. 里程碑 M2（已完成）：阶段 C 核心执行链路完成。
3. 里程碑 M3（已完成）：阶段 D + 阶段 E 完成产品闭环。
4. 里程碑 M4（已完成）：阶段 F + 阶段 G + 阶段 H 发布。

## 6. 下一步执行清单（Beta 反馈期）

1. 收集 Beta 用户反馈并按阻断级别分流。
2. 优先修复发布阻断缺陷，维持每周发布节奏。
3. 评估持久化与调度能力升级（SQLite/Redis 可选）。
4. UI 多页面重构已完成，执行报告：`docs/UI_REFACTOR_EXEC_REPORT.md`。
